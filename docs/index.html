What it is

Simulation-first closed-loop research

pancri.ai is a proof-of-concept platform for studying autonomous glucose control algorithms before any patient contact. Run it, break it, learn from it.

🧬

Physiological model

Bergman minimal model for glucose–insulin kinetics. Meal absorption, insulin-on-board tracking, and dawn-phenomenon drift.

⚙️

Closed-loop controller

Basal rate modulation, dual-wave bolus delivery, and correction logic — the full autonomous loop running every simulation tick.

📊

Clinical reporting

Time-in-range, peak glucose, hypo events, insulin delivery summary — formatted the way an endocrinologist expects to read it.

🔬

Profile sweeps

Run the same scenario across a cohort of simulated patient profiles. Spot systemic risks that single-run testing misses.

📋

Retrospective analysis

Paste in CGM data and the platform will replay what the controller would have done — side-by-side with the real trace.

🛡️

Explainability layer

Every decision gate annotated. See exactly why the controller increased basal, withheld a bolus, or triggered a safety clamp.

How it works

Scenarios

Six clinical stress tests

Each scenario is designed to expose a specific failure mode in the control algorithm.

Baseline meal Standard 60 g carb bolus — sanity-check for time-in-range
Missed bolus Meal without pre-bolus — how far does the correction loop recover?
Dawn phenomenon 3 AM cortisol-driven glucose rise with no meal event
Exercise hypoglycemia Insulin sensitivity spike mid-activity — clamp test
Late correction Correction bolus delayed 90 min — stacking risk assessment
Basal deficit Under-programmed overnight basal — sustained drift above range
Safety

Three layers, no exceptions

The safety architecture is enforced at every simulation tick. The controller cannot deliver insulin outside these bounds regardless of the scenario configuration.

🔴

Hard clamps

Absolute maximum bolus (10 U), maximum basal rate (3× programmed), and minimum glucose floor (60 mg/dL) before insulin delivery halts.

🟡

IOB accounting

Insulin-on-board is tracked continuously. Correction boluses are reduced or withheld when stacking risk exceeds threshold.

🟢

Rate-of-change guard

Rapid glucose drop (>2 mg/dL/min) suspends basal delivery and flags a predictive low alert before the hypo occurs.

⚠ Research use only

pancri.ai is a simulation research platform. It does not connect to any insulin pump, continuous glucose monitor, or patient system. No real insulin is delivered. No patient data is collected or stored. This platform is not a medical device and is not intended for clinical decision-making. All scenarios use synthetic physiological parameters.
