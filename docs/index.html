What it is

Simulation-first closed-loop research

pancri.ai is a proof-of-concept platform for studying autonomous glucose control algorithms before any patient contact. Run it, stress-test it, learn from it.

🧬

Physiological model

Bergman minimal model for glucose–insulin kinetics. Meal absorption, insulin-on-board tracking, and dawn-phenomenon drift.

⚙️

Closed-loop controller

Cause-aware dosing: MEAL / BASAL_DRIFT / REBOUND / FLAT detection, dual-wave bolus, rate-of-rise tiered micro-bolus, and online ISF learning — the full autonomous loop every 5 minutes.

📊

Clinical reporting

Time-in-range, peak glucose, hypo events, insulin delivery summary — formatted the way an endocrinologist expects to read it.

🔬

3-State Arming Gate

Doctor-specified Gate 0: MONITORING → ARMED → FIRING. Insulin only releases when a genuine excursion is confirmed — not sensor noise or transient spikes.

📋

Retrospective analysis

Paste in CGM data and the platform replays what the controller would have done — side-by-side with the real trace.

🛡️

Explainability layer

Every decision gate annotated. See exactly why the controller increased basal, withheld a bolus, or triggered a safety clamp.

How it works

From scenario to report in seconds

No setup, no patient data required.

1

Choose a scenario

Select from nine pre-built clinical challenges spanning meals, hypo, basal drift, exercise, overnight, and stacked corrections. Run all nine at once in Clinical Review mode.

2

Run the simulation

The closed-loop controller runs the full scenario window. Each delivered dose feeds back into the physiology model — glucose responds to what the algorithm actually gave.

3

Read the report

ADA/EASD-scored verdict (PASS/FAIL/SAFE): time-in-range ≥70%, peak glucose <250 mg/dL, 0 hypo steps. Full clinical metrics: TIR, average CGM, glycemic variability, total insulin delivered.

4

Inspect every decision

The Controller Decision Log shows every timestep: CGM reading, arming phase, safety gate that fired, recommended dose vs delivered dose, IOB, suspension state. Export to CSV.

Scenarios

Nine clinical stress tests

Each scenario targets a specific failure mode. All nine are scored automatically against ADA/EASD targets (TIR ≥70%, peak <250 mg/dL, 0 hypo steps).

Overnight Stability Low-stimulation overnight window — verifies the controller stays quiet when glucose is stable

Safety

Eight gates, every tick

Eight independent safety gates run in sequence at every 5-minute CGM step. The controller cannot deliver insulin unless all gates pass. Every gate intervention is named, logged, and auditable.

Gate 0 — Arming

3-State Arming Gate (doctor-specified)

MONITORING (slope 0.3–0.5) → ARMED (≥0.5 for 5 min) → FIRING (≥0.7 for 10 min or +5 mg/dL cumulative rise). Any drop >3 mg/dL/min or negative slope resets to MONITORING immediately.

Gates 1–5 — Guards

Hypo suspension + stateless guards

Stateful hypo suspension (holds until glucose recovers above threshold + 10 mg/dL margin and is rising). Trend confirmation, hypoglycemia prediction, and IOB stacking guard run at every step.

Gates 6–7 — Delivery

Per-interval cap + pass-through

Maximum dose per 5-minute interval enforced. If a recommendation exceeds the cap it is clipped, not blocked. A full pass-through confirms all gates cleared and delivers in full.

Research use only

pancri.ai is a simulation research platform. It does not connect to any insulin pump, continuous glucose monitor, or patient system. No real insulin is delivered. No patient data is collected or stored. This platform is not a medical device and is not intended for clinical decision-making.
